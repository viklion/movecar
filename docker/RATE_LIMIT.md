# IP限流功能说明

## 功能概述

已在挪车应用中实现了完整的IP限流机制，防止单个IP滥用通知服务。**默认情况下限流功能不启用**，可通过环境变量灵活配置。

## 限流规则

### 默认配置（不限制）
- **5分钟内通知次数**：无限制（`RATE_LIMIT_5MIN=0`）
- **每日通知次数**：无限制（`RATE_LIMIT_DAILY=0`）

### 自定义配置示例

#### 场景1：启用5分钟限制
```bash
RATE_LIMIT_5MIN=2      # 5分钟内最多2次
RATE_LIMIT_DAILY=0     # 无每日限制
```

#### 场景2：同时启用两种限制
```bash
RATE_LIMIT_5MIN=2      # 5分钟内最多2次
RATE_LIMIT_DAILY=3     # 每天最多3次（凌晨00:00:00自动重置）
```

#### 场景3：只限制每日请求
```bash
RATE_LIMIT_5MIN=0      # 无5分钟限制
RATE_LIMIT_DAILY=5     # 每天最多5次
```

## 环境变量配置

### RATE_LIMIT_5MIN
- **类型**：整数
- **默认值**：0（不限制）
- **含义**：单个IP在5分钟内最多发送的通知次数
- **配置方式**：在 `docker-compose.yaml` 中设置
- **示例**：`RATE_LIMIT_5MIN=2`

### RATE_LIMIT_DAILY
- **类型**：整数
- **默认值**：0（不限制）
- **含义**：单个IP在一天内最多发送的通知次数
- **重置时间**：每天凌晨00:00:00自动重置
- **配置方式**：在 `docker-compose.yaml` 中设置
- **示例**：`RATE_LIMIT_DAILY=3`


## 用户提示信息

当触发限流时，用户将看到以下提示：

### 5分钟限制触发
- **图标**：⏱️
- **标题**：「操作过于频繁」
- **描述**：「请5分钟后再试」

### 每日限制触发
- **图标**：🚫
- **标题**：「超出每日限制」
- **描述**：「您今天已发送过多次通知」

## Docker容器重启行为

当Docker容器重启时：
- ✅ 所有IP限流记录自动清空
- ✅ 用户可立即重新开始发送通知
- ✅ 新的统计周期开始计算

## 监控和调试

### 查看当前配置
容器启动时会在日志中显示配置信息。

### 常见场景

**场景1：想要完全禁用限流**
```bash
RATE_LIMIT_5MIN=0
RATE_LIMIT_DAILY=0
# 或者不设置这两个变量，因为默认值就是0
```

**场景2：严格的防滥用模式**
```bash
RATE_LIMIT_5MIN=1    # 5分钟内最多1次
RATE_LIMIT_DAILY=2   # 每天最多2次
```

**场景3：宽松模式（允许相对自由的使用）**
```bash
RATE_LIMIT_5MIN=5    # 5分钟内最多5次
RATE_LIMIT_DAILY=10  # 每天最多10次
```
